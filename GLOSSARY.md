# torch_brain Context

Core framework for neural population decoding and processing from electrophysiological recordings.

## Language

### Core Data Structures

**Session**:
A single data collection period, represented by a single H5 file. It can contain multiple tasks and does not strictly have to be continuous.
*Avoid*: Recording, run

**Brainset**:
A curated, standardized collection of neural and behavioral data resulting from a specific publication or experimental source, generated by a dedicated pipeline and structured as a collection of Session files.
*Avoid*: Dataset, corpus, brainset pipeline

**Brainset Pipeline**:
The functional codebase or execution script triggered during data preparation to download raw source data and programmatically transform it into standardized Session files.
*Avoid*: Preprocessor, brainset, data loader script, ingestor

**Dataset**:
A PyTorch-compatible data loading interface provided for each Brainset, used to sample and feed Session data into a model during training. It is typically a subclass of :class:`torch.utils.data.Dataset` and is designed to be used with PyTorch DataLoaders.
*Avoid*: Dataloader, corpus

**Domain**:
The total temporal boundaries over which a neural signal, behavioral stream, or recording is defined and holds valid data. It may consist of a single contiguous span or a union of disjoint time intervals.
*Avoid*: Time span, recording duration, timeline

**Interval**:
A set of one or more time ranges, each defined by a start time and an end time. May carry additional per-range attributes (e.g. trial metadata). Used both to define Domains and to represent collections of experimental epochs such as trials.
*Avoid*: Epoch, segment, time range

**Data**:
A top-level container object that aggregates diverse neural, behavioral, and experimental attributes under a single unified temporal Domain. May seamlessly nest other Data objects or specialized series containers, serving as the canonical programmatic block representing a complete Session slice or trial window.
*Avoid*: Dictionary, record, payload, state map

**ArrayDict**:
A dictionary-like container of multi-dimensional arrays or tensors that strictly share a uniform first dimension. Permits individual attributes to carry distinct inner dimensions and data types while ensuring rigorous row-wise index synchronization.
*Avoid*: Dataframe, table, matrix, structural dictionary

**RegularTimeSeries**:
A uniform data container for dense sequential observations sampled at a fixed, regular frequency. Automatically anchors its temporal boundaries to a sample grid based on its sampling rate, enforcing that the first dimension represents sequentially advancing, evenly spaced time points.
*Avoid*: Matrix, raw array, continuous signal, continuous track

**IrregularTimeSeries**:
An event-driven data container for non-uniformly sampled or discrete sequential observations. Defined explicitly by a 1-D array of ascending timestamps paired with arbitrary attributes that must rigidly share the same first dimension as the event axis.
*Avoid*: Point process, event tracker, sparse tensor, spike list

### Signals & Hardware

**Unit**:
A source of discrete neural events (spikes). This includes both isolated single neurons and unsorted multi-unit activity (threshold crossings).
*Avoid*: Neuron, cell

**Channel**:
A single 1D time-series stream of data within an array (e.g., a row in an EEG matrix). A channel has a specific modality/type but does not have to be limited to brain data (e.g., stimulus triggers, eye tracking).
*Avoid*: Stream, track, trace

**Device**:
The physical hardware used to record the data (e.g., Neuropixels probe, Utah array, EEG cap).
*Avoid*: Probe, hardware

**Modality**:
The semantic meaning and structural specification of a given Channel, Unit, or output. It dictates the data type (e.g., continuous, discrete), dimensions, and appropriate loss function (e.g., `cursor_velocity_2d`, `lfp`, `eeg`, `spikes`).
*Avoid*: Feature type, signal type

### Time Segmentation & Manipulation

**Trial**:
An experimentally defined, behaviorally relevant segment of time (e.g., a stimulus presentation or a reaching movement).
*Avoid*: Epoch, sweep

**Window**:
An arbitrary, fixed-length time slice generated by the framework for batching and training uniform sequences.
*Avoid*: Crop, frame

**Slicing**:
The extraction of a specific `[start, end)` time range from a :class:`Data` object, which can be done either deterministically to isolate fixed trial blocks or stochastically to generate random segments for data training diversity.
*Avoid*: Cropping, chopping, segmenting, patches

**Binning**:
The process of aggregating sparse discrete events within defined time windows into contiguous, fixed-width intervals to produce a dense, uniformly sampled regular time series array.
*Avoid*: Downsampling, digitizing, quantizing

### Model Architecture

**Task**:
The high-level behavioral paradigm or objective the subject is performing during data collection (e.g., "Center-Out Reach", "Sleep").
*Avoid*: Paradigm, experiment

**Readout**:
The specific neural network head or projection layer that maps the model's latent embeddings to a target predicted variable (e.g., a velocity readout, a state classifier).
*Avoid*: Decoder, predictor

**Token**:
A single computational element consisting of a flat array or tensor of data points prepared for model ingestion, completely decoupled from any strict input matrix shape or dimensions (the exact structural attributes of which are model-specific).
*Avoid*: Matrix row, node, vector point

**Tokenizer**:
A component or callable interface that takes a raw `Data` object as input and maps its underlying signals, dimensions, and experimental intervals into the exact unstructured token array format required for model consumption.
*Avoid*: Preprocessor, parser, collator, data encoder

**Latent**:
A learned, structural token used by the model to query or compress information across time, not tied to a specific biological unit or hardware channel.
*Avoid*: Query token, memory token

### Operations & Evaluation Lifecycle

**Transform**:
A callable operation that takes a `Data` object, modifies it (either deterministically or stochastically), and returns it.
*Avoid*: Augmentation, preprocessor

**Stitching**:
The process of aggregating and reconciling model predictions across overlapping time windows to reconstruct a single continuous prediction stream.
*Avoid*: Merging, pooling, aggregating

**Padding**:
A batching strategy where variable-length sequences are extended with filler values to match the longest sequence, resulting in uniform `(B, T, ...)` tensors.
*Avoid*: Masking (when referring to the batching strategy itself)

**Chaining**:
A batching strategy where variable-length sequences are concatenated along the time dimension into a single flat sequence `(Total_T, ...)`, avoiding padding overhead.
*Avoid*: Concatenating (as a formal strategy name), flattening

**Split**:
A specific partition of data designated for a distinct phase of the machine learning lifecycle (strictly "train", "val", or "test").
*Avoid*: Subset, group, bucket, tier

**Split Strategy**:
The underlying systemic logic or axis used to segregate time blocks or identifiers into training, validation, and testing partitions, determined solely by structural boundaries (strictly "intrasession" for slicing a single continuous span, "intersession" for dividing recording blocks, or "intersubject" for isolating different biological subjects).
*Avoid*: Split type, validation regime, fold method, partition shape

**Regime**:
A predefined, rigid benchmark configuration or evaluation setting that hardcodes explicit constraints on which specific subject identities or sessions are permitted to participate in training versus testing phases (e.g., forcing a designated subject to serve exclusively as a fixed training anchor while evaluating on a separate target subject).
*Avoid*: Task type, testing condition, split strategy

**Fold**:
A single, self-contained iteration of train, validation, and test assignments generated deterministically from a Split Strategy for cross-validation.
*Avoid*: Partition, split index

**Assignment**:
The deterministic mapping of a specific Session, subject, or fold partition to a designated Split (strictly "train", "val", or "test") within a cross-validation loop.
*Avoid*: Allocation, labeling, placement, split mapping
