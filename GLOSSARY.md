# torch_brain Context

End-to-end Deep Learning framework for creating training pipelines and models for neural data.

## Language

### Core Data Structures

**Session**:
A single data collection period, represented by a single H5 file. It can contain multiple Tasks and does not strictly have to be continuous. Its in-memory representation is a Data object.
*Avoid*: Recording, run

**Brainset**:
A curated, standardized collection of neural and behavioral data resulting from a specific publication or experimental source, generated by a Brainset Pipeline and structured as a collection of Session files. Each Brainset has a corresponding Dataset that provides programmatic access to its Sessions.
*Avoid*: Dataset, corpus, brainset pipeline

**Brainset Pipeline**:
The functional codebase or execution script triggered during data preparation to download raw source data and programmatically transform it into standardized Session files for a given Brainset.
*Avoid*: Preprocessor, brainset, data loader script, ingestor

**Dataset**:
A PyTorch-compatible data loading interface provided for each Brainset, used to sample and feed Session data into a model during training. Exposes Sampling Intervals as its contract with Samplers. It is typically a subclass of :class:`torch.utils.data.Dataset` and is designed to be used with PyTorch DataLoaders.
*Avoid*: Dataloader, corpus

**Domain**:
The total temporal boundaries over which a RegularTimeSeries, IrregularTimeSeries, or Session is defined and holds valid data. Represented as an Interval, it may consist of a single contiguous span or a union of disjoint time ranges.
*Avoid*: Time span, recording duration, timeline

**Data**:
A container object that aggregates diverse neural, behavioral, and experimental attributes under a single unified Domain. May nest other Data objects or specialized series containers (RegularTimeSeries, IrregularTimeSeries, Interval, ArrayDict), serving as the canonical programmatic representation of a complete Session or a Window sliced from one.
*Avoid*: Dictionary, record, payload, state map

**Interval**:
A set of one or more time ranges, each defined by a start time and an end time. May carry additional per-range attributes (e.g. trial metadata). Used to define Domains, represent collections of Trials, and specify Sampling Intervals.
*Avoid*: Epoch, segment, time range

**ArrayDict**:
A dictionary-like container of multi-dimensional arrays or tensors that strictly share a uniform first dimension. Permits individual attributes to carry distinct inner dimensions and data types while ensuring rigorous row-wise index synchronization. Typically used within Data to store per-Unit or per-Channel attributes.
*Avoid*: Dataframe, table, matrix, structural dictionary

**RegularTimeSeries**:
A uniform data container for dense sequential observations sampled at a fixed, regular frequency. Each instance has its own Domain. Used within Data to represent continuous signals such as LFP or EEG from one or more Channels.
*Avoid*: Matrix, raw array, continuous signal, continuous track

**IrregularTimeSeries**:
An event-driven data container for non-uniformly sampled or discrete sequential observations. Defined explicitly by a 1-D array of ascending timestamps paired with arbitrary attributes sharing the same first dimension. Each instance has its own Domain. Used within Data to represent spike events from Units or irregularly sampled behavioral signals.
*Avoid*: Point process, event tracker, sparse tensor, spike list

### Signals & Hardware

**Unit**:
A putative neuron isolated from a multi-neuron electrophysiological recording, typically using a spike-sorting algorithm. It is a source of discrete neural evens (spikes). A Session's spike events are stored as an IrregularTimeSeries with a per-spike reference to the originating Unit. Metadata about a Session's units is stored as an ArrayDict within the Session's Data object.  
*Avoid*: Neuron, cell

**Channel**:
A source of 1D time-series data stream within an array (e.g., a row in an EEG matrix). Each Channel has a specific Modality but does not have to be limited to brain data (e.g., stimulus triggers, eye tracking). Channel data is typically stored as a RegularTimeSeries within a Data object.  
*Avoid*: Stream, track, trace

**Modality**:
The semantic meaning and structural specification of a given Channel, Unit, or output. It dictates the data type (e.g., continuous, discrete), dimensions, and appropriate loss function if used as a Readout target (e.g., `cursor_velocity_2d`, `lfp`, `eeg`, `spikes`).
*Avoid*: Feature type, signal type

### Time Segmentation & Manipulation

**Trial**:
An experimentally defined, behaviorally relevant segment of time (e.g., a stimulus presentation or a reaching movement). Represented as an Interval within a Session's Data object.
*Avoid*: Epoch, sweep, interval, chunk, experiment

**Window**:
An arbitrary, fixed-length time slice produced by a Sampler for batching and training uniform sequences. A Window is a contiguous time range of data that is typically used for training a model. It can match with a Trial, be a random subset of a Trial, or be a fixed-length window not aligned with any Trial. Slicing a Session by a Window yields a Data object.
*Avoid*: Crop, frame, slice, patch, segment, interval, chunk

**Sampler**:
A component that generates indices (session ID + time range) defining which Windows or Trials to load from a Dataset. Samplers consume Sampling Intervals to know where they are permitted to draw from and produce indices that the Dataset uses to Slice the underlying Sessions.
*Avoid*: Iterator, index generator

**Slicing**:
The process of extracting a specific `[start, end)` time range from a Data object, producing a new Data object for that Window. Can be done deterministically (to isolate Trials) or stochastically (to generate random Windows for training diversity).
*Avoid*: Cropping, chopping, segmenting, patches

### Operations & Evaluation Lifecycle

**Transform**:
A callable operation that takes a Data object, modifies it (either deterministically or stochastically), and returns it. Transforms are applied after Slicing and before tokenization.
*Avoid*: Augmentation, preprocessor

**Padding**:
A batching strategy where variable-length Token sequences are extended with filler values to match the longest sequence in the batch, resulting in uniform `(B, T, ...)` tensors.
*Avoid*: Masking (when referring to the batching strategy itself)

**Chaining**:
A batching strategy where variable-length Token sequences are concatenated along the first dimension into a single flat sequence `(Total_T, ...)`, avoiding Padding overhead.
*Avoid*: Concatenating (as a formal strategy name), flattening

**Split**:
A specific partition of data designated for a distinct phase of the machine learning lifecycle (strictly `"train"`, `"val"`, or `"test"`).
*Avoid*: Subset, group, bucket, tier

**Split Strategy**:
The underlying systemic logic or axis used to segregate time blocks or identifiers into Splits, determined solely by structural boundaries (strictly "intrasession" for slicing a single continuous span, "intersession" for dividing Sessions, or "intersubject" for isolating different biological subjects). A Split Strategy produces one or more Folds.
*Avoid*: Split type, validation regime, fold method, partition shape

**Sampling Intervals**:
A dictionary mapping Session identifiers to Interval objects that define the temporal regions from which a Sampler is permitted to draw Windows or Trials. Serves as the primary contract between a Dataset and a Sampler, specifying where in time sampling is allowed for a given Split.
*Avoid*: Valid regions, eligible windows, sample space

**Fold**:
A single, self-contained iteration of train, validation, and test Assignments generated deterministically from a Split Strategy for cross-validation.
*Avoid*: Partition, split index

**Assignment**:
The deterministic mapping of a specific Session, subject, or fold partition to a designated Split (strictly `"train"`, `"val"`, or `"test"`) within a cross-validation loop.
*Avoid*: Allocation, labeling, placement, split mapping
